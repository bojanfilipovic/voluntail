# Architecture Decision Records (ADRs)

Place ADRs here, e.g. `0001-title-in-kebab-case.md`. Use a short **Context / Decision / Consequences** (and **Status**: Accepted, Superseded) and link to relevant code paths. Reference `git` commits or PRs when useful.

**When to add an ADR:** a decision that affects how the system is **structured**, **deployed**, **secured**, or **exposed to clients** (API, env) — or when you deliberately **defer** something (e.g. SEO) with a one-line “revisit when …”

See also: [`../prompts/VOLUNTAIL_PROMPT.md`](../prompts/VOLUNTAIL_PROMPT.md) — update the prompt in sync when the decision is part of a closed planning / shipping step.
